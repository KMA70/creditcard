1. Overview

The Credit Card Approval System stores applicant information, their financial profiles, submitted applications, and the ML model's prediction outcomes. The diagram below describes the four core entities and their relationships.

3. Entity Descriptions

APPLICANT

Stores personal and demographic information for each credit card applicant.

ColumnTypeConstraintDescription
PKapplicant_idINTNOT NULL, AUTO_INCREMENTUnique applicant identifier
first_nameVARCHAR(60)NOT NULLApplicant's first name
last_nameVARCHAR(60)NOT NULLApplicant's last name
genderCHAR(1)NOT NULLGender code: a=Male, b=Female
ageDECIMAL(5,2)NOT NULLAge in years (e.g. 35.50)
marriedCHAR(1)NOT NULLMarital status: u=Unmarried, y=Married, l=Other
citizenCHAR(1)NOT NULLCitizenship: g=ByBirth, p=ByOtherMeans, s=Temporary
zip_codeVARCHAR(10)NOT NULLResidential zip / postal code
created_atTIMESTAMPDEFAULT NOW()Record creation timestamp

FINANCIAL_PROFILE

Captures the financial attributes of an applicant used as features for the ML model.

ColumnTypeConstraintDescription
PKprofile_idINTNOT NULL, AUTO_INCREMENTUnique profile identifier
FKapplicant_idINTNOT NULL, REFERENCES APPLICANTOwning applicant
debtDECIMAL(10,2)NOT NULLExisting debt in USD
incomeDECIMAL(12,2)NOT NULLAnnual income in USD
credit_scoreINTNOT NULLApplicant's credit score
years_employedDECIMAL(5,2)NOT NULLYears in current/last employment
employedBOOLEANNOT NULLCurrently employed flag
prior_defaultBOOLEANNOT NULLPrior credit default flag

APPLICATION

Represents a single credit card application submission by an applicant.

ColumnTypeConstraintDescription
PKapplication_idINTNOT NULL, AUTO_INCREMENTUnique application identifier
FKapplicant_idINTNOT NULL, REFERENCES APPLICANTApplicant who submitted
FKprediction_idINTREFERENCES PREDICTIONAssociated ML prediction
submitted_atTIMESTAMPDEFAULT NOW()Submission timestamp
statusVARCHAR(20)NOT NULLpending / approved / rejected / under_review

PREDICTION

Stores the output of the ML model for each applicant prediction request.

ColumnTypeConstraintDescription
PKprediction_idINTNOT NULL, AUTO_INCREMENTUnique prediction identifier
FKapplicant_idINTNOT NULL, REFERENCES APPLICANTApplicant for whom prediction was made
model_nameVARCHAR(60)NOT NULLName of the ML model used
approvedBOOLEANNOT NULLTrue if model predicts approval
prob_approvedDECIMAL(5,4)NOT NULLProbability of approval (0–1)
prob_rejectedDECIMAL(5,4)NOT NULLProbability of rejection (0–1)
confidenceVARCHAR(10)NOT NULLConfidence level: High / Moderate / Low
predicted_atTIMESTAMPDEFAULT NOW()Time prediction was generated

4. Relationship Summary

From EntityRelationshipTo EntityDescription
APPLICANT1 : 1FINANCIAL_PROFILEEach applicant has exactly one financial profile
APPLICANT1 : NAPPLICATIONAn applicant can submit multiple applications over time
APPLICANT1 : NPREDICTIONAn applicant can receive multiple predictions across applications
APPLICATIONN : 1PREDICTIONEach application is linked to one prediction result
