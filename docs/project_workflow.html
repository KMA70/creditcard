2. Detailed Phase Descriptions

1
Data Collection & Loading
  • Source: UCI Machine Learning Repository — Credit Approval dataset (crx.data).
  • 690 records with 15 features and 1 binary target (+ approved, - rejected).
  • Feature names are anonymised for privacy. Column names are assigned during loading: Gender, Age, Debt, Married, BankCustomer, EducationLevel, Ethnicity, YearsEmployed, PriorDefault, Employed, CreditScore, DriversLicense, Citizen, ZipCode, Income.
  • Missing values encoded as ? are replaced with NaN during CSV read.
df = pd.read_csv("data/crx.data", header=None, names=COLUMN_NAMES, na_values="?")
2
Data Pre-processing
  • Target column mapped: + → 1 (Approved), - → 0 (Rejected).
  • ZipCode treated as a categorical string (not a numeric value).
  • Numeric missing values imputed with the median strategy.
  • Categorical missing values imputed with the most frequent (mode) strategy.
3
Feature Engineering
  • Numeric features (Age, Debt, YearsEmployed, CreditScore, Income): Standardised using StandardScaler (zero mean, unit variance).
  • Categorical features (Gender, Married, BankCustomer, EducationLevel, Ethnicity, PriorDefault, Employed, DriversLicense, Citizen, ZipCode): Encoded using OneHotEncoder with handle_unknown="ignore".
  • A ColumnTransformer combines both pipelines into a single pre-processing step.
4
Model Training

Four classification algorithms are trained inside a unified scikit-learn Pipeline (preprocessor + classifier):

AlgorithmKey Hyperparameters
Logistic Regressionmax_iter=1000, random_state=42
Random Forestn_estimators=200, random_state=42
Gradient Boosting (XGBoost-style)n_estimators=200, learning_rate=0.1, max_depth=4, random_state=42
Decision Treemax_depth=6, random_state=42
  • Data split: 80% training / 20% testing with stratified sampling (random_state=42).
  • Train size: 552 samples  |  Test size: 138 samples.
5
Model Evaluation

Each model is evaluated using three metrics:

MetricDescriptionPrimary?
AccuracyFraction of correct predictions over the test setSecondary
ROC-AUCArea under the Receiver Operating Characteristic curve — measures discrimination abilityPrimary
CV ROC-AUC5-fold cross-validation ROC-AUC on the training set — guards against overfittingSecondary

A full classification report (precision, recall, F1) and confusion matrix are printed to console for each model.

6
Model Selection
  • The model with the highest test ROC-AUC is automatically selected as the best model.
  • In typical runs Random Forest wins with ROC-AUC ≈ 0.93.
  • Selection is fully automated — no manual intervention required.
7
Model Serialisation
  • The full sklearn Pipeline (preprocessor + classifier) is pickled to models/best_model.pkl.
  • A metadata dictionary is pickled to models/model_meta.pkl containing: model name, ROC-AUC, column order, and all results.
  • Both artefacts are required by the Flask application and the export script.
with open("models/best_model.pkl", "wb") as f:
    pickle.dump(best_pipeline, f)
8
Local Web Application Deployment
  • Flask loads the pickled model at startup — no retraining needed on each request.
  • The web UI (/) displays the model comparison table and an applicant input form.
  • POST to /predict returns an HTML result page with approval decision, probability bars, confidence badge, and risk guidance.
  • POST to /api/predict returns a JSON response for programmatic integration.
  • GET /health returns a liveness probe JSON (useful for process monitors).
  • No cloud account, API key, or external network connection is required.
python app.py
# → Serving on http://localhost:5000
